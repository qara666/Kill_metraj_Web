# Исправление обработки Excel файлов через Render

## Проблема
При обработке Excel файла через Render показывалось 0 заказов из-за недоступности backend сервера.

## Решение
Добавлена fallback обработка Excel файлов прямо во frontend:

### 1. Создан ExcelService для frontend
- Файл: `/frontend/src/services/ExcelService.ts`
- Обрабатывает Excel файлы с помощью библиотеки `xlsx`
- Автоматически определяет колонки по заголовкам
- Создает структурированные данные (заказы, курьеры, способы оплаты)

### 2. Обновлен API сервис
- Файл: `/frontend/src/services/api.ts`
- Добавлен fallback механизм
- Сначала пытается отправить файл на backend
- При ошибке переключается на frontend обработку
- Показывает понятные сообщения об ошибках

### 3. Установлена зависимость
```bash
npm install xlsx
```

## Как работает
1. Пользователь загружает Excel файл
2. Frontend пытается отправить файл на `https://killmetraj-backend.onrender.com/api/upload/excel`
3. Если backend недоступен, файл обрабатывается во frontend
4. Результат отображается с пометкой "(fallback режим)"

## Тестирование
1. Запустите frontend: `npm run dev`
2. Откройте http://localhost:5173
3. Загрузите Excel файл с заказами
4. Проверьте, что заказы обрабатываются даже без backend

## Формат Excel файла
Поддерживаемые заголовки колонок:
- Номер заказа / Order Number / ID
- Клиент / Customer / Имя
- Адрес / Address
- Телефон / Phone
- Курьер / Courier
- Способ оплаты / Payment
- Сумма / Amount / Цена
- Статус / Status
- Дата доставки / Date
