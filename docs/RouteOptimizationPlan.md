# План по оптимизации маршрутов курьеров

## Цель
Автоматизировать процесс разбивки заказов курьеров на маршруты, чтобы упростить вычисление километража и обновление карточек курьеров.

## Основные атрибуты модели Order
- **orderNumber**: уникальный номер заказа.
- **address**: адрес доставки.
- **courier**: идентификатор курьера, которому назначен заказ.
- **status**: статус заказа (в процессе, завершен и т.д.).
- **assignedDate**: дата назначения заказа.
- **completedDate**: дата завершения заказа.
- **estimatedDeliveryTime**: предполагаемое время доставки.
- **actualDeliveryTime**: фактическое время доставки.

## Задачи
1. **Получение заказов курьера**:
   - Фильтрация заказов по полю `courier` и статусу.
   
2. **Разделение на маршруты**:
   - Создание маршрутов с количеством заказов от 1 до 7, основанное на времени назначения и завершения.

3. **Вычисление километража**:
   - Использование адресов для вычисления расстояний между начальной и конечной точками маршрута через API геокодирования.

4. **Обновление карточки курьера**:
   - Интеграция данных о пройденном километраже и количестве заказов в карточку курьера.

## Ожидаемые результаты
- Упрощение работы с заказами и маршрутизацией.
- Уменьшение необходимости в ручной обработке и повышение эффективности.

## Следующие шаги
1. Разработка функционала для получения и обработки данных.
2. Внедрение расчета километража.
3. Обновление данных в карточке курьера.
