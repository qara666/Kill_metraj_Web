# Swagger API Integration - User Guide

## Обзор

Интеграция Swagger API позволяет загружать данные о заказах и курьерах напрямую из вашей системы управления заказами, минуя ручную загрузку Excel файлов.

## Быстрый старт

### 1. Получение API ключа

Свяжитесь с администратором системы для получения API ключа. Ключ имеет формат:
```
KMFh23298&23201)8172^&21Vs@gfgsd
```

### 2. Загрузка данных

1. Откройте страницу **AutoPlanner**
2. Найдите секцию "Загрузка данных"
3. Нажмите кнопку **"Загрузить из Swagger API"** (фиолетовая кнопка с иконкой облака)

![Swagger Import Button](./swagger-import-button.png)

### 3. Настройка параметров

В открывшемся модальном окне заполните следующие поля:

#### Обязательные параметры:

**API Ключ** 🔑
- Введите ваш API ключ
- Ключ сохраняется автоматически для последующих загрузок
- Хранится безопасно в локальном хранилище браузера

**Начало окна доставки** 🕐
- Выберите дату и время начала периода доставки
- Формат: `dd.mm.yyyy HH:MM`
- По умолчанию: текущая дата, 11:00

**Конец окна доставки** 🕐
- Выберите дату и время окончания периода доставки
- Формат: `dd.mm.yyyy HH:MM`
- По умолчанию: текущая дата, 23:00

#### Опциональные параметры:

**ID Подразделения** 🏢
- Укажите ID подразделения для фильтрации заказов
- Например: `100000052`
- Оставьте пустым для загрузки всех заказов

### 4. Загрузка

1. Нажмите кнопку **"Загрузить"**
2. Дождитесь завершения загрузки (обычно 1-3 секунды)
3. Данные автоматически обработаются и отобразятся в интерфейсе

## Что происходит после загрузки?

### Автоматическая обработка

1. **Трансформация данных**
   - Заказы преобразуются во внутренний формат
   - Время (kitchenTime, deliverBy) конвертируется в timestamps
   - Курьеры добавляются в список доступных

2. **Геокодирование адресов** 🗺️
   - Все адреса автоматически геокодируются
   - Получаются координаты для каждого заказа
   - Неудачные адреса помечаются для ручной проверки

3. **Готовность к планированию**
   - Заказы готовы для автоматического планирования маршрутов
   - Можно применять фильтры по зонам, курьерам, времени
   - Доступна визуализация на карте

## Формат данных

### Заказы

Каждый заказ содержит:
- **orderNumber** - Номер заказа
- **address** - Адрес доставки
- **kitchenTime** - Время готовности на кухне
- **deliverBy** - Крайний срок доставки
- **deliveryZone** - Зона доставки
- **courier** - Назначенный курьер
- **amount** - Сумма заказа
- **paymentMethod** - Способ оплаты

### Курьеры

Каждый курьер содержит:
- **name** - Имя курьера
- **isActive** - Активен ли курьер
- **vehicleType** - Тип транспорта (car/foot)

## Кеширование

Система автоматически кеширует загруженные данные:
- **Время жизни кеша**: 5 минут
- **Хранение**: 10 минут после последнего использования
- **Повторная загрузка**: Автоматически при изменении параметров

Это означает, что повторные загрузки с теми же параметрами будут мгновенными!

## Производительность

### Метрики загрузки

В консоли разработчика вы увидите:
```

⏱️ Загрузка из Swagger API заняла 1234.56ms
✅ Загружено 150 заказов и 12 курьеров
```

### Оптимизация

- Используйте фильтр по подразделению для уменьшения объема данных
- Сужайте временное окно для более быстрой обработки
- Кеш автоматически ускоряет повторные загрузки

## Устранение неполадок

### Ошибка: "API ключ не предоставлен"
**Решение**: Убедитесь, что вы ввели API ключ в соответствующее поле

### Ошибка: "Swagger API недоступен"
**Решение**: 
1. Проверьте интернет-соединение
2. Убедитесь, что сервер Swagger API работает
3. Свяжитесь с администратором

### Ошибка: "Параметр dateShift обязателен"
**Решение**: Убедитесь, что выбрана дата в полях времени доставки

### Медленная загрузка
**Возможные причины**:
- Большое количество заказов (>500)
- Медленное интернет-соединение
- Высокая нагрузка на сервер

**Решение**: Используйте фильтр по подразделению или сузьте временное окно

### Не все адреса геокодированы
**Это нормально!** Некоторые адреса могут быть некорректными или неполными.
- Проверьте адреса вручную
- Используйте фильтр для исключения проблемных заказов
- Обновите адреса в исходной системе

## Советы и рекомендации

### Ежедневное использование

1. **Утренняя загрузка**
   - Загружайте данные на весь день (11:00-23:00)
   - Используйте кеш для быстрого доступа

2. **Обновление в течение дня**
   - Загружайте данные каждые 30-60 минут
   - Фокусируйтесь на текущем временном окне

3. **Планирование маршрутов**
   - После загрузки используйте фильтры
   - Группируйте заказы по зонам
   - Назначайте курьеров автоматически

### Безопасность

- **Не делитесь API ключом** с посторонними
- API ключ хранится только в вашем браузере
- Очистка кеша браузера удалит сохраненный ключ

### Интеграция с Excel

Вы можете комбинировать оба метода:
1. Загрузите основные данные из Swagger API
2. Дополните расписанием курьеров из Excel
3. Используйте все возможности планирования


```
x-api-key: YOUR_API_KEY
```

## Часто задаваемые вопросы

**Q: Сколько заказов можно загрузить за раз?**
A: Максимум 2000 заказов. Рекомендуется не более 500 для оптимальной производительности.

**Q: Как часто обновляются данные в Swagger API?**
A: Данные обновляются в реальном времени. Рекомендуется загружать каждые 30-60 минут.

**Q: Можно ли загружать данные за прошлые даты?**
A: Да, просто выберите нужную дату в полях времени доставки.

**Q: Что делать, если забыл API ключ?**
A: Свяжитесь с администратором системы для получения нового ключа.

**Q: Работает ли интеграция офлайн?**
A: Нет, требуется интернет-соединение для доступа к Swagger API.
