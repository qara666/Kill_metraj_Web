# Интеграция с Telegram для парсинга сообщений

## Описание

Реализована полная интеграция с Telegram API для парсинга сообщений из групп и чатов. Система позволяет искать сообщения по текстовым запросам, включая семизначные номера.

## Установка зависимостей

### Backend

```bash
cd backend
npm install
```

Библиотека `telegram` (gramjs) уже добавлена в `package.json`.

### Frontend

Зависимости уже установлены, дополнительных действий не требуется.

## Настройка

### 1. Получение API ID и API Hash

1. Перейдите на https://my.telegram.org/apps
2. Войдите в свой аккаунт Telegram
3. Создайте новое приложение или используйте существующее
4. Скопируйте:
   - **API ID** (число, например: 12345678)
   - **API Hash** (строка, например: abc123def456...)

### 2. Использование

1. Откройте страницу "Парсинг выгрузки в телеграм и реестре" в приложении
2. Нажмите "Подключиться"
3. Введите:
   - API ID
   - API Hash
   - Номер телефона (в формате +380XXXXXXXXX)
4. Нажмите "Подключиться"
5. Если требуется авторизация:
   - Введите код подтверждения из Telegram
   - Нажмите "Подтвердить"
6. После подключения:
   - Выберите чаты/группы для поиска
   - Введите запрос (можно с семизначными номерами)
   - Нажмите "Начать поиск"

## Функционал

### Поиск сообщений

- **По семизначным номерам**: Автоматически извлекает все семизначные номера из запроса и ищет их во всех выбранных чатах
- **По тексту**: Если в запросе нет семизначных номеров, выполняется поиск по тексту
- **Фильтрация чатов**: Можно выбрать конкретные чаты/группы для поиска
- **Типы чатов**: Поддержка групп, каналов и личных сообщений

### API Endpoints

#### Backend

- `POST /api/telegram/initialize` - Инициализация подключения
- `POST /api/telegram/complete-auth` - Завершение авторизации с кодом
- `GET /api/telegram/status/:sessionId` - Проверка статуса подключения
- `GET /api/telegram/chats/:sessionId` - Получение списка чатов
- `POST /api/telegram/search/:sessionId` - Поиск сообщений
- `POST /api/telegram/disconnect/:sessionId` - Отключение

## Структура файлов

### Backend

- `backend/src/services/TelegramService.js` - Основной сервис для работы с Telegram
- `backend/src/routes/telegramRoutes.js` - API роуты
- `backend/simple_server.js` - Интеграция роутов в сервер
- `backend/sessions/` - Директория для хранения сессий (создается автоматически)

### Frontend

- `frontend/src/services/telegramService.ts` - Клиентский сервис для работы с API
- `frontend/src/pages/TelegramParsing.tsx` - Компонент страницы парсинга

## Безопасность

- Сессии хранятся локально на сервере в директории `backend/sessions/`
- API ID и API Hash не передаются на внешние серверы
- Каждая сессия имеет уникальный ID
- Сессии сохраняются между перезапусками сервера

## Ограничения

- Требуется стабильное интернет-соединение
- Telegram может ограничивать частоту запросов
- Поиск выполняется последовательно по каждому номеру/чату
- Максимальное количество результатов: 100 на запрос (настраивается)

## Устранение неполадок

### Ошибка подключения

Если не удается подключиться к Telegram, проверьте следующее:

1. **Правильность API ID и API Hash:**
   - Убедитесь, что вы используете правильные значения с https://my.telegram.org/apps
   - API ID должен быть положительным числом (например: 12345678)
   - API Hash должен быть строкой из 32 шестнадцатеричных символов (0-9, a-f)
   - Убедитесь, что при копировании API Hash не попали лишние пробелы или символы

2. **Формат номера телефона:**
   - Номер должен быть в международном формате без пробелов
   - Для Украины: `+380XXXXXXXXX` или `380XXXXXXXXX` (без плюса)
   - Номер должен содержать от 10 до 15 цифр
   - Не должен начинаться с 0

3. **Интернет-соединение:**
   - Проверьте, что у вас есть доступ к интернету
   - Если Telegram заблокирован в вашей стране, может потребоваться VPN или прокси
   - Проверьте, что серверы Telegram доступны (попробуйте открыть web.telegram.org)

4. **Проблемы с сессией:**
   - Если старая сессия не работает, удалите файлы из папки `backend/sessions/`
   - Попробуйте подключиться заново с новым номером телефона

5. **Таймауты:**
   - Если подключение занимает слишком много времени, проверьте настройки файрвола
   - Убедитесь, что порты для подключения к Telegram не заблокированы

### Ошибка авторизации

1. Убедитесь, что код подтверждения введен правильно
2. Код действителен ограниченное время, запросите новый
3. Проверьте, что номер телефона правильный

### Чаты не загружаются

1. Проверьте статус подключения
2. Убедитесь, что авторизация завершена успешно
3. Проверьте логи сервера на наличие ошибок

### Сообщения не находятся

1. Убедитесь, что выбран хотя бы один чат
2. Проверьте правильность запроса
3. Убедитесь, что в выбранных чатах есть сообщения с таким содержимым
