# Исправления Excel Processor - 06.01.2026

## Проблема
Ручная загрузка Excel файлов не распознавала заказы (возвращала 0 заказов).

## Корневая причина
В файле `frontend/src/utils/data/excelProcessor.ts` было **три разных функции `isValidAddress`** с несогласованной логикой:

1. В `processJsonData` (строка 338) - основной сканер
2. В `createOrder` (строка 791) - создание заказа из `isOrderRow`
3. В `createOrderFromData` (строка 1295) - создание заказа из `findOrderNumber`

Проблемы:
- Разные минимальные длины адреса (5 vs 10 символов)
- Неполный список ключевых слов для "явных адресных колонок"
- Отсутствие паттерна для адресов типа "Ленина 5" (номер в конце)
- Строгая проверка `excludeColumns` даже для колонок с названием "Адрес"

## Решение

### Синхронизированы все три функции `isValidAddress`:

1. **Минимальная длина**: `< 3` символа (было 5-10)
2. **Явные адресные колонки** (пропускают строгую валидацию):
   - address, адрес, addr
   - куда, доставка, delivery
   - улица, street
   - место, location
   - пункт, point

3. **Улучшенные паттерны адресов**:
   - Добавлен: `/\b\d+[а-яa-z]?$/` - номер в конце ("Ленина 5")
   - Расширены маркеры улиц: str, street, пер., переулок
   - Расширены маркеры деталей: під'їзд, л/с, кл, apartment, habteka

4. **Логика проверки**:
   ```typescript
   if (isExplicitAddressColumn && hasText) {
       if (isNotPhone && isNotEmail && isNotOnlyNumber) {
            return true; // Пропускаем проверку маркеров
       }
   }
   ```

5. **Минимальная длина текста**: `> 2` символа (было 5-10)

## Файлы изменены
- `/frontend/src/utils/data/excelProcessor.ts` (3 функции синхронизированы)

Теперь система должна корректно распознавать:
- ✅ Простые адреса: "Ленина 5", "вул Хрещатик 10"
- ✅ Адреса с явными колонками: "Адрес доставки", "Куда везти"
- ✅ Короткие адреса: минимум 3 символа
- ✅ Адреса без явных маркеров в колонках с названием "адрес"
