# Предложения по улучшению автопланирования маршрутов

## Текущие проблемы

1. **Последовательный greedy алгоритм** - добавляет заказы по очереди, не оптимально
2. **Нет кластеризации** - не группирует близкие адреса
3. **Медленный расчет** - каждый кандидат требует полный запрос к Directions API
4. **Нет предварительной оценки** - нельзя быстро отсеять неподходящие заказы
5. **Фиксированный порядок** - не оптимизируется внутри маршрута после добавления

## Предложенные улучшения

### 1. Кластеризация заказов по близости (приоритет: ВЫСОКИЙ)
- Группировать заказы в кластеры по географической близости
- Использовать k-means или иерархическую кластеризацию
- Преимущества: меньше запросов к API, более логичные маршруты

### 2. Быстрая предварительная фильтрация (приоритет: ВЫСОКИЙ)
- Использовать Haversine distance для грубой оценки расстояния
- Отсеивать заказы, которые явно не подходят по времени/расстоянию
- Только "перспективные" кандидаты проверять через Directions API
- Преимущества: ускорение в 5-10 раз

### 3. Оптимизация порядка заказов внутри маршрута (приоритет: СРЕДНИЙ)
- После формирования базового маршрута применять локальный поиск
- Проверять перестановки соседних заказов (2-opt)
- Если новый порядок лучше и соблюдает дедлайны - использовать его
- Преимущества: сокращение дистанции на 10-20%

### 4. Умная оценка кандидатов (приоритет: СРЕДНИЙ)
- Оценка "сложности" добавления заказа: расстояние, время, дедлайн
- Приоритет заказам с близкими дедлайнами и адресами
- Бонус за заказы в том же районе/на той же улице
- Преимущества: более логичные маршруты

### 5. Кэширование Directions API (приоритет: НИЗКИЙ)
- Кэшировать результаты Directions для повторяющихся пар адресов
- Использовать в памяти кэш на время сессии планирования
- Преимущества: ускорение при повторных запросах

### 6. Параллельная обработка (приоритет: НИЗКИЙ)
- Проверять несколько кандидатов параллельно (Promise.all)
- Выбирать лучший из успешных результатов
- Преимущества: быстрее для больших наборов заказов

## Рекомендуемый план реализации

### Фаза 1: Быстрые улучшения (1-2 часа)
1. Добавить предварительную фильтрацию через Haversine
2. Улучшить приоритет кандидатов (близость + дедлайн)

### Фаза 2: Оптимизация маршрутов (2-3 часа)
3. Применить локальный поиск (2-opt) для улучшения порядка
4. Добавить кластеризацию для больших наборов (>20 заказов)

### Фаза 3: Дополнительные оптимизации (опционально)
5. Кэширование Directions API
6. Параллельная обработка

## Метрики для оценки

- **Время планирования** - должно уменьшиться в 3-5 раз
- **Качество маршрутов** - сокращение дистанции на 10-20%
- **Количество запросов к API** - уменьшение на 40-60%
- **Удовлетворение дедлайнов** - должно остаться на том же уровне или улучшиться
