# Структурированная система обработки Excel файлов

## 🎯 Цель
Переделать систему так, чтобы после загрузки Excel файла все данные структурировались по связям:
- **Номер заказа** → **Курьер** + **Адрес**
- **Способ оплаты** → **Сумма к оплате**

## ✅ Реализованная структура

### 1. Структурированные заказы
Каждый заказ теперь содержит логически сгруппированные данные:

```javascript
{
  id: "9086195",
  status: "Исполнен",
  type: "Доставка",
  customer: {
    name: "Денис Дзюба",
    phone: "+380501234567",
    totalOrders: 5,
    comment: "VIP клиент"
  },
  address: {
    full: "Київ, вул. Сергія Данченка, 32",
    comment: "Зустріч з кур'єром на заправці ОККО",
    zone: "Зона 1 Ближние",
    deliveryTime: "15мин"
  },
  timing: {
    created: "22.09.2025 08:55",
    kitchen: "09:00",
    deliverBy: "11:00",
    planned: "12:00",
    total: "45мин"
  },
  financial: {
    amount: 679,
    discount: 0,
    change: 0,

  },
  courier: "Негода Юрій",
  comment: "GLOVO: 101436402710, CODE: 028",
  coordinates: null,
  geocoded: false
}
```

### 2. Группировка по курьерам
Каждый курьер содержит:
- Список всех своих заказов
- Общую сумму и количество заказов
- Зоны работы
- Используемые способы оплаты

```javascript
{
  name: "Негода Юрій",
  orders: [
    {
      id: "9086195",
      customer: "Денис Дзюба",
      address: "Київ, вул. Сергія Данченка, 32",
      amount: 679,
      status: "Исполнен",
      type: "Доставка"
    }
  ],
  totalAmount: 929,
  orderCount: 2,
  zones: ["Зона 1 Ближние"],
  paymentMethods: ["Готівка"]
}
```

### 3. Группировка по способам оплаты
Каждый способ оплаты содержит:
- Список всех заказов с этим способом оплаты
- Общую сумму и количество заказов
- Среднюю сумму заказа

```javascript
{
  method: "Готівка",
  orders: [
    {
      id: "9086195",
      customer: "Денис Дзюба",
      amount: 679,
      status: "Исполнен"
    }
  ],
  totalAmount: 2549,
  orderCount: 3,
  averageAmount: 849.67
}
```

### 4. Группировка по адресам
Каждый адрес содержит:
- Список всех заказов по этому адресу
- Общую сумму и количество заказов
- Зоны доставки
- Курьеров, работающих по этому адресу

```javascript
{
  address: "Київ, вул. Сергія Данченка, 32",
  orders: [
    {
      id: "9086195",
      customer: "Денис Дзюба",
      courier: "Негода Юрій",
      amount: 679,
      status: "Исполнен"
    }
  ],
  totalAmount: 929,
  orderCount: 2,
  zones: ["Зона 1 Ближние"],
  couriers: ["Негода Юрій"]
}
```

## 🔗 Связи данных

### 1. Номер заказа → Курьер + Адрес
- Каждый заказ привязан к конкретному курьеру
- Каждый заказ имеет конкретный адрес
- Курьеры группируются по заказам и адресам
- Адреса группируются по заказам и курьерам

### 2. Способ оплаты → Сумма к оплате
- Каждый способ оплаты содержит все заказы с этим способом
- Рассчитывается общая сумма по каждому способу оплаты
- Рассчитывается средняя сумма заказа по каждому способу

## 🎯 Преимущества новой структуры

1. **Логическая группировка** - данные сгруппированы по смыслу
2. **Легкий доступ** - можно быстро найти все заказы курьера или по адресу
3. **Аналитика** - встроенная статистика по всем группам
4. **Связи** - четкие связи между заказами, курьерами, адресами и оплатой
5. **Масштабируемость** - легко добавлять новые группировки
