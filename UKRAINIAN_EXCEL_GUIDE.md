# 🇺🇦 Гід по роботі з Excel файлами в Kill Metraj

## 📋 Огляд функціональності

Система тепер підтримує повну обробку Excel файлів з автоматичним розбиттям на блоки даних та сопоставленням номерів замовлень з способами оплати та кур'єрами.

## 🔧 Підтримувані формати файлів

- **Excel файли**: `.xlsx`, `.xls`
- **CSV файли**: `.csv`
- **Максимальний розмір**: 10MB

## 📊 Структура Excel файлу

### Обов'язкові колонки для замовлень:
- **Номер замовлення** (або `Order Number`, `orderNumber`)
- **Адреса** (або `Address`, `address`)

### Додаткові колонки:
- **Телефон** (або `Phone`, `phone`)
- **Кур'єр** (або `Courier`, `courier`)
- **Спосіб оплати** (або `Payment`, `paymentMethod`)
- **Сума** (або `Amount`, `price`)
- **Примітка** (або `Note`, `comment`)
- **Пріоритет** (або `Priority`, `priority`)
- **Статус** (або `Status`, `status`)
- **Дата** (або `Date`, `date`)
- **Час** (або `Time`, `time`)

## 🎯 Автоматичне розбиття на блоки

Система автоматично розпізнає та розділяє дані на:

### 1. **Замовлення** (Orders)
- Містить номер замовлення та адресу
- Автоматично прив'язується до кур'єра та способу оплати
- Зберігається в базі даних з повною інформацією

### 2. **Кур'єри** (Couriers)
- Створюються автоматично на основі колонки "Кур'єр"
- Дублікати ігноруються
- Оновлюється статистика виконання

### 3. **Способи оплати** (Payment Methods)
- Створюються автоматично на основі колонки "Спосіб оплати"
- Підтримують різні типи: готівка, карта, онлайн, банківський переказ
- Налаштування комісій та обмежень

### 4. **Маршрути** (Routes)
- Автоматично створюються для кожного кур'єра
- Групування замовлень за кур'єрами
- Оптимізація маршрутів (в майбутньому)

## 🚀 Процес обробки

### 1. **Завантаження файлу**
- Виберіть Excel файл через інтерфейс
- Система перевірить формат та розмір
- Покаже попередній перегляд даних

### 2. **Автоматичне розпізнавання**
- Система проаналізує заголовки колонок
- Визначить тип кожного рядка (замовлення, кур'єр, спосіб оплати)
- Згрупує дані за типами

### 3. **Валідація даних**
- Перевірка обов'язкових полів
- Валідація форматів даних
- Перевірка унікальності номерів замовлень

### 4. **Збереження в базі даних**
- Створення кур'єрів (якщо не існують)
- Створення способів оплати
- Збереження замовлень з прив'язкою до кур'єрів та способів оплати
- Створення маршрутів для кожного кур'єра

### 5. **Генерація звіту**
- Кількість оброблених замовлень
- Кількість створених кур'єрів
- Кількість способів оплати
- Список помилок та попереджень

## 📊 Результат обробки

Після успішної обробки ви отримаєте:

### **Замовлення**
- Всі замовлення з файлу зберігаються в базі
- Автоматично прив'язуються до кур'єрів та способів оплати
- Можна переглядати, редагувати та відстежувати статус

### **Кур'єри**
- Автоматично створюються нові кур'єри
- Оновлюється статистика виконання
- Можна призначати додаткові замовлення

### **Способи оплати**
- Створюються нові способи оплати
- Налаштовуються комісії та обмеження
- Використовуються для аналітики

### **Маршрути**
- Автоматично створюються для кожного кур'єра
- Групування замовлень за кур'єрами
- Оптимізація послідовності доставки

## 🎯 Переваги нової системи

1. **Автоматичне розпізнавання** - система сама визначає тип даних
2. **Гнучкість** - підтримка різних форматів заголовків
3. **Валідація** - перевірка даних перед збереженням
4. **Зв'язки** - автоматичне сопоставлення замовлень з кур'єрами та оплатою
5. **Звітність** - детальна інформація про обробку
6. **Українська мова** - повна локалізація інтерфейсу
