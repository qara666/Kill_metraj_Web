# Отслеживание данных Дашборда на Render (Единый процесс)

Для совместимости с бесплатным планом Render и упрощения деплоя, система была переведена на **единый процесс**. Теперь API-сервер и фоновый загрузчик данных (Fetcher) работают вместе.

## 1. Как теперь это работает

Больше не нужно запускать отдельные воркеры. При запуске основного сервера (`node simple_server.js` или через `npm start`):
1. **API Server**: Запускается на порту 5001.
2. **Dashboard Fetcher**: Автоматически запускается внутри того же процесса.
3. **Logs**: Все логи (и от сервера, и от загрузчика) теперь выводятся в консоль Render (`stdout`), даже в режиме production.

## 2. Проверка работы в панели Render

1. Откройте вкладку **Logs** вашего сервиса.
2. Вы должны увидеть сообщения:
   - `[kill-metraj-backend]: Dashboard Fetcher started within main process` — подтверждает запуск загрузчика.
   - `[kill-metraj-backend]: Фоновый загрузчик дашборда` — заголовок логов фетчера.
   - `[kill-metraj-backend]: Сохранено X заказов (Yмс)` — подтверждает успешное получение данных из Fast API.

## 3. Если данные все еще не приходят

Проверьте следующие переменные в **Environment** на Render:
- `EXTERNAL_API_KEY`: Ваш ключ для API Fastopertor.
- `EXTERNAL_API_URL`: `http://app.yaposhka.kh.ua:4999/api/v1/dashboard`
- `NODE_ENV`: `production`

## 4. Диагностика (Debug)
Если в логах пусто или есть ошибки, используйте наш инструмент диагностики:
Откройте в браузере: `https://yapiko-auto-km-backend.onrender.com/api/debug/fetcher`
*(Нужна авторизация Bearer Token)*

Она покажет:
- **schema**: Есть ли нужные колонки в БД.
- **latest_records**: Когда были последние обновления.
- **external_api**: Доступно ли внешнее API из сети Render.
